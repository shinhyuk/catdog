# CLAUDE.md — 개냥 대전 (Dog vs Cat)

> 이 파일은 저장소 루트(`/CLAUDE.md`)에 둔다. Claude Code가 매 세션 읽는다.

## 프로젝트 개요

GPS 기반 출퇴근형 모바일 PvP 게임 “개냥 대전”. 걸으면 자산이 쌓이고, 개 진영 vs 고양이 진영이 서로 약탈하며 겨룬다. 최종 목표는 iOS 네이티브 앱이지만, **지금은 GitHub Pages 정적 웹 프로토타입으로 핵심 루프의 “느낌”부터 검증**하는 단계다.

설계 원본은 `docs/`에 있다. 게임 규칙을 바꾸거나 새로 정할 때는 **코드와 함께 `docs/`도 갱신**한다.

- `docs/game-spec.md` — 전체 컨셉·경제·진영
- `docs/skilltree.md` — 스킬트리 전체 (시즌1 / 만렙 20)

-----

## 현재 목표 — Phase 1 (이 단계에서만 집중)

**GitHub Pages에 올라가는, 혼자서 핵심 루프를 끝까지 돌려볼 수 있는 플레이 가능한 웹 프로토타입.**

검증할 것: 걷기 → 레벨업·자산 적립 → 매일 아침 직업 선택 → 호구 명단 보고 약탈 → 털리고 복수 → 스킬트리 찍기. 이 루프가 “재밌는가”만 본다.

실제 GPS, 실제 멀티플레이, 백엔드 서버, 결제, 네이티브 빌드는 **전부 Phase 1 범위 밖**(아래 “범위 밖” 참조). 다른 플레이어 자리는 **봇으로 시뮬레이션**한다.

-----

## 기술 스택 & 핵심 제약

- **Vite + React + TypeScript + Tailwind CSS**
- **정적 배포 only** — GitHub Pages. 서버 사이드 코드 없음, 런타임 백엔드 없음
- **상태 저장 = `localStorage`** (브라우저). DB·API 호출 없음
- **걸음수 = 실제 이동 또는 시뮬.** (Phase 1.5) `navigator.geolocation`으로 실제 이동거리를 걸음으로 환산한다. GPS 거부/PC에선 “걷기” 버튼 시뮬로 폴백. 실제 만보기(가속도계) 카운팅은 여전히 범위 밖.
- **실제 거리 지도 = MapLibre GL + OpenFreeMap.** (Phase 1.5) 키 불필요 무료 벡터 타일로 실제 길을 3D 기울여(pitch) 표시. 나침반(`DeviceOrientation`)으로 바라보는 방향 = 지도 회전(bearing). HTTPS 필수(GitHub Pages·localhost OK).
- **시간 정산(매일 06:00) = 시뮬.** “다음 날로” 버튼으로 빠르게 하루 넘기기 가능하게
- **봇 = 클라이언트에서 시뮬레이션.** 공개 NPC(야생 들개/길고양이), 수동 8 : 능동 2
- **비밀키·토큰 금지.** 클라이언트 코드는 전부 공개됨

## GitHub Pages 함정 (반드시 지킬 것)

1. **base path:** 프로젝트 페이지(`user.github.io/repo`)이므로 `vite.config.ts`에 `base: '/<repo-name>/'` 필수. **배포 후 흰 화면이면 90% 이게 원인.**
1. **SPA 라우팅:** GitHub Pages는 서버 라우팅이 없다. `HashRouter` 사용(가장 안전). 또는 `404.html` 리다이렉트 트릭.
1. **배포:** `dist/`를 GitHub Actions로 Pages에 publish. `gh-pages` 수동 푸시 대신 Actions 워크플로 권장.
1. **에셋 경로:** 절대경로(`/img.png`) 대신 import 또는 base 상대경로 사용.

## 핵심 게임 규칙 (요약 — 상세는 `docs/`)

**진영 (영구 소속):** 🐕 개 / 🐈 고양이. Phase 1은 무료 선택. *유료 변경·글로벌 진영경제는 범위 밖.*

**자원 2종 (역할 분리):**

- 걸음수 = **경험치** → 레벨업 전용. **시즌1 만렙 = Lv 20**
- 약탈로 뺏는 것 = **자산** → 뺏고 뺏김
- **어제 걸음 = 남에게 보이는 자산 규모** / **오늘 걸음 = 실제로 털리는 지갑**
- 약탈은 피해자의 **오늘 걸음에서 누수(leak)**로 차감 (즉시 X, 그 사람이 걷는 동안 정해진 양 채워질 때까지 야금야금)

**직업 3종 (진영 공통, 매일 아침 변경, 가위바위보 상성):**

- 🔍 **정찰자** — 어제 걸음 읽어 호구를 **진영 공유 명단**에 등록. 2~5% 소액 절도. 은신 간파 + 노출 1시간 시각 지정. *약점: 직접 벌이가 약함, 본인 무방비*
- 💥 **강탈자** — 명단/확인 타겟 **10% 강탈**. 약탈 후 본인 노출. 은신자 못 뚫음, 호구 스스로 못 찾음(정찰 의존). 정찰자에게 **커미션 1~3%** 지급
- 🥷 **은신형** — 명단에서 사라짐. 절도 0(적립만). 간파당하면 노출(본인에게 통보, **60~30분**)
- 상성: **정찰 > 은신 > 강탈 > (노출) > 정찰**

**스킬트리:**

- 레벨 = **각 직업 트리마다 독립 지급되는 스킬포인트** (Lv20 → 세 직업에 각각 20)
- 직업당 3갈래. 갈래마다 13pt(스킬 12 + 궁극기 1)
- 게이트: 중단(T2)=갈래 5pt / 궁극기(T3)=갈래 12pt
- **단단한 칸막이** → 만렙 20에서 직업당 궁극기 1개만. 전체 데이터는 `docs/skilltree.md`
- **삼각 상성 천장은 스킬 풀업해도 안 깨진다** (정찰 절도 5% 상한, 강탈 10% 천장은 콤보로만 +5%p, 은신은 강탈이 영원히 못 뚫음 등)

**봇 (Phase 1 핵심):** 공개 NPC = 야생 들개/길고양이. 수동 8(먹잇감) : 능동 2(날 털어 복수 루프 유발). 솔로 테스트의 밀도를 봇이 채운다.

**시즌:** 종료 시 레벨·스킬포인트·랭크 리셋. 진영·계정은 유지.

## 코딩 규칙

- **모든 밸런스 수치는 `src/config/balance.ts` 상수로.** 절도율·노출시간·커미션·봇 비율·게이트 등 게임에 들어가는 숫자를 컴포넌트나 로직에 하드코딩하지 않는다. 튜닝이 곧 게임 디자인이다.
- **게임 로직은 순수 함수**로 짜서 UI와 분리한다(`src/game/`). 입력→출력만 있게 해서 단위 테스트 가능하게.
- **직업·스킬·진영은 데이터 주도**(`src/data/`). 새 스킬 추가가 데이터 한 줄이 되도록. `skilltree.md` 구조를 그대로 타입으로 옮긴다.
- 상태는 한 곳(`src/state/`)에서만 `localStorage`와 동기화. 컴포넌트가 직접 localStorage 만지지 않게.

## 작업 방식 (Claude Code)

- 큰 변경(새 화면·새 시스템) 전에는 **plan mode**로 계획부터 확인받는다.
- **PR은 항상 자동 생성한다(사용자 상시 승인).** 작업을 작업 브랜치에 푸시하면 별도 요청 없이 `main` 대상 PR을 만든다.
- 커밋은 작은 단위, conventional commits (`feat:`, `fix:`, `chore:`).
- 게임 규칙을 바꾸면 **`docs/`의 해당 스펙도 같은 커밋에서 갱신**한다.
- 막히면 임의로 범위를 늘리지 말고, Phase 1 DoD 기준으로 최소 동작부터 만든다.

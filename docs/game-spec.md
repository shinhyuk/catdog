# 개냥 대전 — 게임 스펙 (game-spec)

> 설계 원본. 게임 규칙을 바꾸면 코드(`src/`)와 함께 이 문서도 같은 커밋에서 갱신한다.
> Phase 1(웹 프로토타입) 기준. 수치의 실제 값은 `src/config/balance.ts`가 단일 출처(SSOT)다.

## 컨셉

GPS 기반 출퇴근형 모바일 PvP 게임. 걸으면 자산이 쌓이고, 개 진영 vs 고양이 진영이 서로
약탈하며 겨룬다. Phase 1은 GitHub Pages 정적 웹으로 핵심 루프의 "느낌"을 검증한다.
다른 플레이어 자리는 봇으로 시뮬레이션한다.

핵심 루프: **걷기 → 레벨업·자산 적립 → 매일 아침 직업 선택 → 호구 명단 보고 약탈 →
털리고 복수 → 스킬트리 찍기.**

## 진영 (영구 소속)

- 🐕 개 / 🐈 고양이. Phase 1은 무료 선택, 변경 불가.
- 유료 변경·글로벌 진영경제는 범위 밖.

## 자원 2종 (역할 분리)

| 자원 | 출처 | 용도 |
|------|------|------|
| 경험치(EXP) | 걸음수 | 레벨업 전용. 시즌1 만렙 Lv 20 |
| 자산(Assets) | 걸음 적립 + 약탈 | 뺏고 뺏김. 진영 점수 |

- **어제 걸음 = 남에게 보이는 자산 규모** (`yesterdaySteps`)
- **오늘 걸음 = 실제로 털리는 지갑** (`todaySteps`)
- 약탈은 피해자의 오늘 걸음에서 **누수(leak)**로 차감된다. 즉시 빠지지 않고,
  그 사람이 걷는 동안 정해진 양이 채워질 때까지 야금야금 빠진다.
  (`LEAK_DRAIN_RATE` = 걷는 동안, `LEAK_SETTLE_RATE` = 06:00 정산 시 강제 차감)

## 직업 3종 (진영 공통, 매일 아침 변경)

가위바위보 상성: **정찰 > 은신 > 강탈 > (노출) > 정찰**

### 🔍 정찰자
- 적의 어제 걸음을 읽어 호구를 **진영 공유 명단**에 등록.
- 2~5% 소액 절도 (`SCOUT_MIN`~`SCOUT_MAX`, 상한 `SCOUT_CAP`=5%).
- 은신자 **간파** + 노출 1시간(`SCOUT_REVEAL_MIN`) 시각 지정.
- 약점: 직접 벌이가 약하고 본인은 무방비.

### 💥 강탈자
- 명단/확인 타겟 **10% 강탈** (`ROBBER_BASE`).
- 천장 10%, **콤보로만 +5%p**(`ROBBER_COMBO_BONUS`, 복수 연계 시).
- 약탈 후 본인 노출(`ROBBER_SELF_MIN`).
- 은신자 못 뚫음. 호구를 스스로 못 찾아 정찰에 의존.
- 정찰자에게 **커미션 1~3%**(`COMMISSION_MIN`~`COMMISSION_MAX`).

### 🥷 은신형
- 명단에서 사라진다. 절도 0(적립만).
- 간파당하면 노출(본인에게 통보, 60~30분 `STEALTH_DETECTED_*`).
- 강탈은 은신을 영원히 못 뚫는다(천장).

## 약탈 천장 (삼각 상성, 스킬 풀업으로도 안 깨짐)

- 정찰 절도율 5% 상한.
- 강탈 10% 천장, 콤보로만 +5%p.
- 은신은 강탈이 영원히 못 뚫음.

## 봇 (Phase 1 핵심)

- 공개 NPC = 야생 들개 / 길고양이.
- **수동 8 : 능동 2** (`ACTIVE_RATIO`). 능동 봇이 나를 털어 복수 루프를 유발.
- 풀 크기 `POOL_SIZE`. 매일 걸음·직업을 다시 굴린다.

## 하루 사이클 (06:00 정산 시뮬)

"다음 날로" 버튼으로 하루를 넘긴다:
1. 남은 leak을 강제 차감(정산).
2. 오늘 걸음 → 내일의 "보이는 자산"으로 이동, 오늘 지갑 0 리셋.
3. 봇 갱신 + 능동 봇이 보이는 자산을 노려 강탈(새 leak 생성).
4. 직업 선택 해제, 명단 초기화 후 아군 정찰자가 일부를 다시 채움.

## 시즌

종료 시 레벨·스킬포인트·랭크·자산·명단 리셋. 진영·계정은 유지.

## 범위 밖 (Phase 1)

실제 GPS, 실시간 멀티플레이, 백엔드, 글로벌 진영 경제, 결제, 시즌 랭킹 서버,
9개 궁극기 정밀 밸런싱(구조만, 수치는 placeholder), 네이티브 iOS 빌드.
